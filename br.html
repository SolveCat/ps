BAD RETURN - Pełna dokumentacja

Definicja i podstawowe informacje

Bad Return (Nieprawidłowy zwrot) to formularz tworzony przez Problem Solvera w przypadku wykrycia potencjalnego nadużycia ze strony klienta. Dotyczy przypadków wymagających dalszej weryfikacji przez zespoły śledcze.

Typowe sytuacje wymagające Bad Return:

  • Puste opakowanie wysyłkowe
  • Przedmiot zużyty ponad akceptowalne normy
  • Przedmiot niekompletny
  • Przedmiot różniący się od zamawianego
  • Śmieci w opakowaniu
  • Podejrzenie oszustwa

Narzędzia i wartości graniczne

Podstawowe narzędzia:

  • TRex Advanced Detection (kod fcmenu: 1290)
  • GRASS - weryfikacja ceny produktu
  • RedPhone - dokumentacja fotograficzna (Market Place)
  • FC Research - weryfikacja produktu

Wartości graniczne:

200€ - podstawowy próg cenowy
50€ - próg różnicy cenowej przy Switcheroo/DUSW
300€ - próg dla dodatkowego raportowania mailowego

Proces wystawiania Bad Return

1. Przygotowanie

  • Sprawdź cenę w GRASS
  • Zweryfikuj obecność alertu SYSKA
  • Sprawdź historię w Gravis

2. TRex Advanced Detection

  • Uruchom aplikację (1290)
  • Zeskanuj stanowisko
  • Zeskanuj LPN/RMA
  • Wybierz kategorię
  • Uzupełnij wymagane informacje
  • Dla ≥200€: RedPhone/odpowiedni kod

3. Finalizacja

  • Prawidłowe odkładanie przedmiotu
  • Raportowanie w Rabbit Hole
  • Dodatkowy raport mailowy (>300€)

Przypadki specjalne i raportowanie

Nie wystawiamy Bad Return dla:

  • Gwarancja
  • SYSKA alert
  • Alert Marketplace
  • Problemy z numerem seryjnym przy gwarancji

Specjalna procedura:

  • Uszkodzone baterie - natychmiast do T3/AM
  • Produkty >300€ - raport mailowy
  • Przedmioty z pamięcią - specjalna weryfikacja

Raportowanie:

    • Dokładny opis sytuacji
    • Dodanie numerów LPN/Order ID
  2. RedPhone

    • Odpowiednie oświetlenie
